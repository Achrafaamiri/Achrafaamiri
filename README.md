## 💡 About Me

I'm a software developer studying at **42 Firenze**, driven by one thing: understanding how things really work under the hood.

Before 42, I spent 3 years working in a professional dev team on **Raven ECMS** — an enterprise banking platform used by international financial institutions. That experience showed me how real software is built: Agile workflows, working alongside business analysts, and shipping in production environments.

But I realized I wanted deeper foundations. So I joined **42 Firenze** — no teachers, no lectures, just you, the problem, and your peers. It's the hardest and most honest way to learn programming.

Alongside systems programming, I've been going deep on **AI-assisted development** — not using AI as a shortcut, but learning to orchestrate it properly: writing specs, building pipelines, and always knowing what the code is actually doing.

## 🤖 AI Engineering — Claude Code & Agents

This is where I've invested seriously over the past year. I don't use AI to generate code I don't understand — I use it as a force multiplier when I already know what I want to build.

### **What I've Built**
- **Multi-agent pipelines** with Claude Code: parallel/sequential subagents with specialized roles (Explore, Plan, Execute)
- **Custom Skills** with context-based auto-invocation — pattern matching, progressive disclosure
- **MCP Server integrations**: Playwright, PostgreSQL, GitHub, Notion — full end-to-end automation
- **Slash Commands** for repeatable git-integrated workflows
- **BridgeEye** — a Telegram bot that fetches live crypto market data and uses Claude to reason through technical analysis for users
- **Morpheus** — an AI voice agent integrated into my portfolio site (ElevenLabs + custom prompt engineering), designed to qualify visitors conversationally

### **How I Work With AI**
- Write `CLAUDE.md` files to define project conventions, agent routing, and context rules
- Orchestrate with Opus, execute with Sonnet — cost-aware model selection
- Use Hooks for event-driven automation
- Manage context windows intentionally — no sloppy prompting
